# app_runtime

`app_runtime` 为 `main/app_main.cpp` 提供启动周期中的辅助工具。高层功能流程和组件
初始化顺序保留在 `app_main`，便于直接查看设备在不同唤醒模式下会启动哪些功能。

组件负责：

- 采集并格式化启动上下文；
- 写入复位、唤醒和固件版本诊断；
- 处理启动阶段的异步电池采样回调；
- 输出无线和维护模式状态日志；
- 等待维护接口断开；
- 等待输入及按键事务结束，记录休眠快照并进入深度休眠。

按键、无线、Shell、存储等组件的初始化仍由 `app_main` 明确调用，
`app_runtime` 不隐藏整体启动流程。

## 与 app_main 的边界

`app_main` 保留以下高层流程：

1. 初始化唤醒检测和按键跟踪；
2. 初始化持久化、诊断和电池服务；
3. 判断是否需要启动 ESP-NOW；
4. 区分单次电池按键事务与维护模式；
5. 维护接口断开后停止配对并进入休眠。

`app_runtime` 收口不影响流程阅读的实现细节，例如复位原因文本转换、异步电池回调、
轮询等待、休眠前资源统计和黑匣子同步。

## 日志约束

组件产生的 `ESP_LOG` 消息和写入黑匣子的字符串只使用 ASCII 英文。中文仅用于源码
注释和文档，避免设备输出受到终端编码和日志处理工具的影响。
